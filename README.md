## プロジェクト概要

バニラ JavaScript のみを使用した、野球チケット予約フローの練習用 Web アプリケーションです。
実際の予約サイトに近い UX を再現しながら、高齢者でも迷わず操作できるシンプルな設計を意識して構築しました。

ビルドステップは不要で、HTML / CSS / JS ファイルをそのままブラウザで開くだけで動作します。

デモ: https://baseballticket.netlify.app/html/home.html

## 主な機能

- 試合日程の選択（本日から 4 日間の LG 戦を自動生成）
- 座席エリアの選択（内野席 / 外野席）
- 座席の選択（A・B エリア 5×5 グリッド、最大 4 席）
- 価格確認（チケット合計 + 手数料 3,000 ウォン）
- 配送方法の選択（現地受取 / モバイルチケット / 宅配 +3,000 ウォン）
- カード決済 / 無通帳振込による支払い
- 予約完了画面（予約番号の自動生成・予約内容サマリー）

**ページ遷移フロー**

```
ホーム → 日付選択 → 座席選択 → エリア選択 → 価格確認 → 配送選択 → 決済 → 完了
```

## 工夫した点

### 1. カード決済 UX（高齢者を想定した設計）

このプロジェクトの最大の工夫点です。

一般的なカード決済では、ユーザーが実際のカード番号・有効期限・CVC を手入力します。
しかし高齢者にとってこの操作は煩雑であり、個人情報の入力に対する不安も大きいと考えました。
そのため、以下のアプローチを採用しました。

**ランダムカードの表示方式**

実際のカード情報を入力させるのではなく、システムがランダムに生成した練習用カード（番号・有効期限・CVC）を画面上のカードビジュアルに表示します。
ユーザーはその表示値を見ながら入力欄に転記するだけで決済を完了できます。
実際のカード情報を一切扱わないため、個人情報漏洩のリスクをゼロにしています。

**生年月日による本人確認**

カード情報の入力に加え、配送情報の登録時に入力した生年月日との一致確認を決済時に行います。
認知症や操作ミスによる誤操作を防ぐための補助的なバリデーションとして機能します。

**3 回失敗でカードロック**

入力値が 3 回連続で不一致の場合、決済ボタンをロックします。
繰り返しの入力ミスを抑止する安全装置として組み込んでいます。

**CVC 入力時のカードフリップアニメーション**

CVC 欄にフォーカスが当たると、カードが CSS 3D トランスフォームで裏返り、CVC の位置を視覚的に案内します。
実際にカードの裏面を確認する行為をそのまま UI で再現することで、操作の意味を直感的に伝えます。

---

### 2. 座席選択の設計

複雑な座席マップを避け、学習用として扱いやすいシンプルな構造を採用しました。

- 座席は **A エリア・B エリア** の 2 区画に分け、各エリアに 5×5 = 25 席を配置
- ページ読み込み時に約 20% の座席をランダムで選択不可（満席）状態にすることで、実際の予約サイトに近い状況を再現
- 選択できる座席は最大 4 席まで。上限に達した場合はアラートで通知
- 選択済み座席をクリックすると確認ダイアログを表示し、誤操作によるキャンセルを防止
- 選択した座席番号（A1、B3 など）はリアルタイムで画面に反映され、現在の選択状態を一目で確認できます
